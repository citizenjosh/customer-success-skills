## STRATEGIC OBJECTIVE

Personalize the welcome flow to match the user's specific functional needs.

## DATA INPUTS

- **Signup** role query
- **Team** size data
- **Intent** signals

## EXECUTION LOGIC

1. **Capture** user role and use case during the initial signup step.
2. **Dynamically** adjust the onboarding path to present relevant first steps for that persona.
3. **Ensure** the user reaches their specific "**Aha! Moment**" via a tailored journey.

## GUARDRAILS

- **ENSURE** the tailored journey does **NOT** exclude essential core product features.
- **REQUIRE** personalization to be based on explicit user input or high-confidence intent signals.
- **REQUIRE** all persona paths to include a 'Technical Readiness' check as a prerequisite for value journeys.
