## STRATEGIC OBJECTIVE

Enable users to overcome friction points without manual support intervention.

## DATA INPUTS

- **"Stalled"** session alerts
- **Real-time** event logs

## EXECUTION LOGIC

1. **Detect** "**stalled**" sessions where a user is hovering or repeating actions without progress.
2. **Trigger** contextual hints or guided tours relevant to the specific friction point.
3. **Resolve** user confusion in real-time to maintain onboarding momentum.

## GUARDRAILS

- **ENSURE** hints are helpful and non-intrusive, avoiding "pop-up fatigue."
- **REQUIRE** hints to be **ONLY** triggered when there is a high probability of user confusion.
- **ENSURE** hints are sourced from verified docs and do not advise on security/financial settings without a link.
